# アプリについて

このプロジェクトは、Goで構築されたToDo 管理サービスのAPIです。

## 起動方法

以下の手順でコンポーネントを起動できます。

1. 下記のコマンドでAPIサーバーを起動します。

```sh
docker compose up
```

2. localhost:8080で起動します。


## ディレクトリ構成

```
.
├── Dockerfile
├── Makefile
├── README.md
├── config                    # 設定関連のコードを含むディレクトリ
├── controller                # コントローラー層のディレクトリ
│   ├── request               # APIのリクエストの構造体を定義するディレクトリ
│   ├── response              # APIのレスポンスの構造体を定義するディレクトリ
├── db                        # データベース関連のファイルを含むディレクトリ
│   └── migrations            # マイグレーションファイルを含むディレクトリ
├── docker-compose.yml        # Docker Compose設定ファイル
├── errors                    # エラーハンドリング関連のコードを含むディレクトリ
├── go.mod                    # Goモジュールの依存関係を管理するファイル
├── go.sum                    # 依存関係のバージョン情報を管理するファイル
├── main.go                   # アプリケーションのエントリーポイント
├── middleware                # ミドルウェア関連のコードを含むディレクトリ
├── mock                      # テスト用のモックを含むディレクトリ
├── model                     # ドメインモデルを含むディレクトリ
├── repository                # データアクセス層のコードを含むディレクトリ
├── routes                    # ルーティング設定を含むディレクトリ
└── usecase                   # ビジネスロジック層のコードを含むディレクトリ
```
